Migration Pennylane → IO BILL : le guide en 30 minutes

Publié le 2026-04-22 · Temps de lecture : 8 min · Par OWL'S INDUSTRY

Vous utilisez Pennylane et envisagez de basculer vers IO BILL pour des raisons de coût, de simplicité ou de proximité française ? Ce guide pratique détaille la procédure complète, du téléchargement de vos données Pennylane à la mise en service d'IO BILL. Pour un freelance, comptez 30 minutes ; pour une TPE avec plusieurs années d'historique, 2 heures.

1. Ce que vous récupérez de Pennylane

Pennylane permet l'export de l'ensemble de vos données depuis votre espace, rubrique Paramètres → Export. Les formats disponibles sont :

  • Clients & fournisseurs : CSV avec raison sociale, SIRET, adresse, email, conditions de paiement ;
  • Factures émises : ZIP contenant les PDFs + un CSV récapitulatif (numéro, date, montant, TVA, statut paiement) ;
  • Devis : idem ;
  • Achats : ZIP de justificatifs + CSV avec affectation comptable ;
  • Comptabilité : FEC (Fichier des Écritures Comptables) si vous êtes au régime BNC réel ou BIC réel.

Comptez environ 5 minutes de génération côté Pennylane (l'export est asynchrone et arrive par email).

2. Préparer votre compte IO BILL

Si vous n'avez pas encore de compte IO BILL :

  1. Créez votre compte sur app.iobill.online (essai gratuit 14 jours sans CB) ;
  2. Renseignez votre SIRET — IO BILL pré-remplit automatiquement votre raison sociale, adresse et code NAF via SIRENE ;
  3. Choisissez votre régime fiscal et social (micro-BNC, BIC, BNC réel, etc.) ;
  4. Configurez vos taux de TVA par défaut.

3. Importer vos clients et fournisseurs

IO BILL accepte directement le CSV Pennylane sans transformation. Depuis votre espace IO BILL :

  1. Allez dans Contacts → Importer ;
  2. Sélectionnez votre fichier clients.csv Pennylane ;
  3. IO BILL détecte automatiquement les colonnes et propose un mappage ;
  4. Validez — les contacts sont importés en moins d'une minute (jusqu'à 5 000 contacts).

Les SIRET déjà renseignés dans Pennylane sont automatiquement enrichis par IO BILL avec les informations actualisées (nouveau dirigeant, déménagement, statut administratif) via l'API SIRENE.

4. Importer vos factures émises

Pour conserver la traçabilité historique, IO BILL accepte l'import des factures Pennylane sous forme d'archive :

  1. Allez dans Documents → Import historique ;
  2. Téléversez votre ZIP Pennylane (PDFs + CSV) ;
  3. IO BILL traite chaque PDF, extrait les données, et crée des factures "historisées" non éditables ;
  4. La numérotation est conservée à l'identique pour respecter la continuité.

Les factures historiques apparaissent dans votre journal des ventes mais ne génèrent pas de nouvelles écritures comptables (puisqu'elles sont déjà en compta dans Pennylane). Elles servent uniquement à l'audit et à la consultation.

Astuce : si vous changez de logiciel en milieu d'année, démarrez la nouvelle numérotation IO BILL au numéro suivant la dernière émise dans Pennylane. Par exemple, dernière facture Pennylane = F2026-042 → première IO BILL = F2026-043. La continuité de la numérotation est une obligation légale.

5. Connecter votre banque DSP2

Si vous aviez connecté votre banque dans Pennylane (synchronisation des écritures), refaites le même geste dans IO BILL :

  1. Banque → Connecter un compte ;
  2. Choisissez votre banque dans la liste (250+ établissements supportés) ;
  3. Authentifiez-vous via le tunnel sécurisé Bridge by BPCE ;
  4. Donnez votre consentement DSP2 (90 jours, renouvelable automatiquement).

IO BILL récupère automatiquement les 90 derniers jours d'écritures et les confronte à vos factures importées pour réaliser un auto-lettrage rétroactif.

6. Reconfigurer vos Payment Links Stripe

Si vous utilisiez les Payment Links Pennylane → Stripe pour faire payer vos clients en ligne :

  1. Connectez votre compte Stripe à IO BILL (Paiements → Stripe → Connecter) ;
  2. IO BILL réutilise automatiquement votre clé API Stripe existante ;
  3. Activez l'option "Lien de paiement automatique sur chaque facture" ;
  4. Les nouvelles factures émises depuis IO BILL contiendront un lien Stripe valide.

Les Payment Links déjà émis depuis Pennylane restent valides côté Stripe. Vos clients peuvent continuer à payer leurs anciennes factures sans problème.

7. Résilier Pennylane proprement

Une fois IO BILL opérationnel et vérifié (émission d'au moins 2-3 factures de test), vous pouvez résilier Pennylane :

  • La résiliation se fait depuis votre espace Pennylane, rubrique Abonnement → Résilier ;
  • Pennylane vous laisse 30 jours pour télécharger d'éventuels exports manquants ;
  • Conservez l'archive ZIP complète de Pennylane sur votre disque dur — vous en aurez besoin en cas de contrôle fiscal sur les exercices traités dans Pennylane.

8. Et l'expert-comptable ?

Si vous travaillez avec un expert-comptable, prévenez-le du changement. Il devra :

  • Récupérer son accès à votre espace IO BILL (en mode Cabinet, accès gratuit pour l'expert) ;
  • Recevoir le FEC final Pennylane pour la dernière période ;
  • Réceptionner désormais les exports comptables depuis IO BILL.

Aucun cabinet sérieux ne devrait s'opposer à un changement d'outil. Le format FEC est identique partout (norme DGFiP).

Combien de temps cela prend-il vraiment ?

Pour un freelance solo avec moins de 100 factures historiques : 30 minutes. Pour une TPE avec plusieurs années d'historique et une banque connectée : 1h30 à 2h, étalées sur 2-3 jours pour bien tester chaque étape. Notre support migration peut intervenir en visio si vous le souhaitez (gratuit sur demande à contact@iobill.online).
