v1.0.0 5 mai 2026 Lancement public

🚀 Sortie officielle d'IO BILL 1.0

Après 8 mois de développement et 3 mois de bêta privée avec 30 utilisateurs (freelances, TPE et 2 cabinets pilotes), IO BILL ouvre publiquement aujourd'hui. Le périmètre 1.0 couvre l'intégralité de ce qui était promis sur la landing : facturation Factur-X 2026/2027, banque connectée DSP2, OCR, mode cabinet 50 dossiers, app mobile native iOS & Android.

+ Ajouté
  • Génération de factures Factur-X EN 16931 native (PDF/A-3 + XML CII embarqué)
  • Connexion bancaire DSP2 via Bridge by BPCE (250+ banques compatibles)
  • Auto-lettrage IA des écritures bancaires avec apprentissage continu
  • OCR achats Mistral (illimité plan Pro, 20/mois plan Solo)
  • Mode cabinet : console multi-dossiers (50 inclus), validation TVA, ticketing
  • Export FEC standardisé avec plans Cegid/Sage/ACD/Quadratus/Pennylane
  • Modules TVA (CA3, CA12, franchise) et URSSAF (micro, TNS, assimilé)
  • Apps mobiles natives iOS et Android (App Store et Google Play)
  • API REST publique documentée OpenAPI 3.0 avec webhooks
  • Intégrations Zapier et Make
  • 3 plans tarifaires : Solo 9,90 € / Pro 19,90 € / Cabinet 49 € HT/mois

Roadmap publique

Les fonctionnalités prévues pour les prochaines versions, par ordre de priorité estimée :

  • v1.1 (juin 2026) — Multi-devises (USD, GBP, CHF) avec taux de change BCE quotidien
  • v1.2 (Q3 2026) — Multi-établissements (holding + filiales) sous une connexion unique
  • v1.3 (Q3 2026) — Intégration n8n (alternative auto-hébergée à Zapier)
  • v1.4 (Q4 2026) — Module BTP avancé : situations de travaux multi-tranches, retenue de garantie, attestations URSSAF
  • v2.0 (Q1 2027) — Mode multi-utilisateurs avancé sur le plan Cabinet : permissions par dossier, audit log, archivage probant

Ces dates sont indicatives. Pour suggérer une fonctionnalité ou voter pour celles qui vous intéressent, écrivez à roadmap@iobill.online.

Notification des mises à jour

Pour être notifié des nouvelles versions :

  • Dans l'application : un panneau "Quoi de neuf" s'affiche à chaque connexion après une release majeure ;
  • Email : abonnez-vous à la newsletter sur le blog (envoi à chaque release majeure ou mineure) ;
  • RSS : flux disponible sur /changelog/feed.xml (à venir) ;
  • Plausible custom event : pour les développeurs qui suivent l'API, chaque release est annoncée via un webhook public.
